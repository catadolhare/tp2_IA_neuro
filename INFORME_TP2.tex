\section{Introducción}

El objetivo de este trabajo práctico consistía en entrenar, utilizando algún algoritmo de Aprendizaje por Refuerzo, a un agente para jugar al 4 en línea; juego en el que dos competidores alternadamente.

El objetivo principal fue desarrollar y entrenar una red neuronal profunda que aproxime la función Q óptima, permitiendo al agente tomar decisiones estratégicas en cada estado del juego. Se realizó una búsqueda sistemática de hiperparámetros mediante \textit{grid search} para identificar las configuraciones que maximizan el desempeño del agente.

\section{Decisiones de Arquitectura}

\subsection{Red Neuronal (DQN)}
Se implementó una red neuronal profunda con la siguiente arquitectura:

\begin{itemize}
    \item \textbf{Capa de entrada:} 42 neuronas (tablero $6 \times 7$ aplanado)
    \item \textbf{Capas ocultas:} $256 \rightarrow 256 \rightarrow 128 \rightarrow 64$ neuronas
    \item \textbf{Capa de salida:} 7 neuronas (una por columna)
    \item \textbf{Función de activación:} ReLU en capas ocultas
\end{itemize}

\textbf{Justificación:} La arquitectura profunda con disminución gradual de neuronas permite capturar patrones complejos del juego mientras reduce la dimensionalidad progresivamente. Se optó por una red más profunda que ancha para capturar jerarquías de características estratégicas.

\section{Implementación}

Para facilitar el desarrollo y experimentación del agente, se implementaron dos componentes clave:

\subsection{Sistema de Evaluación}
Se desarrolló un módulo de evaluación (\texttt{evaluar.py}) que permite medir objetivamente el desempeño del agente entrenado. Este sistema enfrenta al agente DQN contra oponentes de control (RandomAgent y DefenderAgent) en un número configurable de partidas, registrando victorias, derrotas y empates. Esta herramienta fue fundamental para comparar diferentes configuraciones de hiperparámetros de manera sistemática.

\subsection{Grid Search Automatizado}
Se implementó un script de grid search completo (\texttt{entrenar\_y\_evaluar\_grid\_search.py}) que automatiza el proceso de entrenamiento y evaluación para todas las combinaciones de hiperparámetros. El sistema:

\begin{itemize}
    \item Entrena un modelo DQN para cada configuración de hiperparámetros
    \item Evalúa automáticamente el desempeño contra RandomAgent (1000 partidas)
    \item Registra los resultados en un archivo CSV para análisis posterior
    \item \textbf{Guarda únicamente el mejor modelo} basándose en el porcentaje de victorias, optimizando el uso de almacenamiento
\end{itemize}

Esta implementación permitió explorar eficientemente el espacio de hiperparámetros y identificar la configuración óptima de manera empírica y reproducible.

\section{Grid Search de Hiperparámetros}

Se realizó una búsqueda exhaustiva sobre \textbf{192 configuraciones diferentes}.

\subsection{Espacio de Búsqueda}
\begin{itemize}
    \item \textbf{Learning rate} ($\alpha$): [0.0001, 0.0005, 0.001, 0.005]
    \item \textbf{Factor de descuento} ($\gamma$): [0.9, 0.95, 0.98, 0.99]
    \item \textbf{Epsilon inicial} ($\epsilon_0$): [0.7, 0.8, 0.9, 1.0]
    \item \textbf{Oponentes de entrenamiento:} [None (auto-juego), RandomAgent, DefenderAgent]
\end{itemize}

\subsection{Parámetros Fijos}
Episodios de entrenamiento: 1000, $\epsilon_{min} = 0.1$, $\epsilon_{decay} = 0.995$, batch size: 128, tamaño de memoria: 1000, actualización de target network: cada 100 pasos.

\section{Conclusiones}

El desarrollo de este trabajo práctico permitió construir un agente capaz de jugar al Connect4 utilizando Deep Q-Learning. Partimos de la definición del ambiente y la representación del estado, acciones y recompensas, y diseñamos una red neuronal profunda para aproximar la función de valor de acción $Q(s,a)$. Mediante entrenamiento con memoria de replay, actualización de red objetivo y política $\epsilon$-greedy, el agente aprendió estrategias efectivas tanto de ataque como de defensa.

Tras probar múltiples configuraciones de hiperparámetros y oponentes a través de un grid search exhaustivo (192 configuraciones), se identificó que: (1) un learning rate relativamente alto ($\alpha=0.005$) es esencial para convergencia efectiva, (2) entrenar contra oponentes con estrategia básica (DefenderAgent) produce mejores políticas que auto-juego o adversarios aleatorios, y (3) factores de descuento medios-altos ($\gamma=0.95-0.98$) balancean adecuadamente recompensas inmediatas y futuras.

El modelo más robusto alcanzó un 92.7\% de victorias frente a agentes aleatorios, demostrando que el aprendizaje se traduce en victorias sostenidas. La evaluación mediante partidas de prueba permitió comprobar la efectividad del agente y confirmar que la combinación de Deep Q-Learning con un diseño cuidadoso del ambiente y de la política de exploración puede generar agentes competentes en juegos estratégicos de dos jugadores.
